通过miniVue浅析vue3核心 ​

本文属于学习中的记录，不一定对，miniVue来源是codewhy课堂学习（侵删）

一、了解vue3 ​

1、渲染模块 ​

vue中有一个h函数，是用于创建虚拟DOM节点（vnode）的,还有一个render函数，是用于编程式地创建组件虚拟 DOM 树的函数，两个函数配合就能替代template的功能，当然render与JSX配合也能实现

js
import { h } from 'vue'

export default {
  data() {
    return {
      msg: 'hello'
    }
  },
  render() {
    return h('div', this.msg)
  }
}

以上代码来自vue官网

2、响应式模块 ​

我们都知道vue3的响应式较于vue2的变化是实现响应式的方法不同，vue2是使用对象的defineProperty实现的，这种方式有几个麻烦的点

  • 无法监听数组的变化
  • 必须遍历对象每个属性
  • 必须深层遍历嵌套的对象
  • 无法监听对象新增的属性，在vue2中是使用**$set**方法来再次监听

而vue3的proxy代理就很好的弥补了上面那些缺点，在vue3中是使用reactive和ref方法来生成响应式数据

  • ref能够代理所有的数据，但是代理的数据只能通过**.value**的方式拿值，因为数据还有包裹些别的属性

  • reactive是只能够代理引用数据类型的数据，因为返回的是直接代理的数据，没有再处理和包裹

当然vue3还有编译模块和别的，但是要实现miniVue知道两个就够了，编译模块比较麻烦

二、miniVue ​

1、h函数 ​

我们通过上面的内容知道h函数是返回一个虚拟DOM节点的（vnode），它有三个参数

  • type标签类型，也可以是组件对象
  • props可选参数，标签上的attribute（属性）
  • children可选参数，子节点

有了这三个参数我们就能生成vonde，那什么是vnode呢

我们能看到h函数返回的是一个对象，对象中有type、props、children，这样我们自己就可以编写一个h函数了

js
/**
 * 生成虚拟dom（js对象）
 * @param {String} tag 节点类型
 * @param {Object} props 虚拟节点的属性
 * @param {String,Array} children 子节点
 * @returns 
 */
const h = (tag, props, children) => {
  return {
    tag,
    props,
    children
  }
}

我们这里简化了h函数，实际的要比这复杂的多，但用于实现简单功能这就够了

2、mount函数 ​

mount函数功能是将虚拟DOM节点转换成真实DOM的，有两个参数vnode、container（真实的node）

由上面实现的h函数我们可以知道，vonde就是一个对象里面有tag(标签)、props(属性)、children(子节点)这三个属性，通过tag我们能够创建一个真实的DOM节点，并且将props设置成其属性

js
const { tag, props, children } = vnode;
// 1.创建真实元素，并且在vnode上保留
const el = vnode.el = document.createElement(tag);
// 2.判断props，设置att
if(props) {
	for(let key in props) {
    const value = props[key];
    if(key.startsWith("on")) {
     el.addEventListener(key.slice(2).toLocaleLowerCase(), value);
    } else {
     el.setAttribute(key, value);
    }
  }
}

完成了上面的事情，我们得到了一个真实的node并成功为其设置了属性，接下来要做的是处理children，分两种情况，第一种是字符串，那我们直接将得到的node的textContent设置成children

js
el.textContent = children

如果是一个vnode那我们递归调用这个处理vnode的mount,只不过container要变成刚得到的node，由于子节点可以是多个，所以是数组，我们这里遍历数组

js
children.forEach(element => {
	mount(element, el)
});

最后就是将node挂载到DOM树上

js
container.appendChild(el);

至此mount函数的功能就已经完成了，完整代码如下

js
/**
 * 挂载虚拟节点
 * @param {Object} vnode 虚拟节点
 * @param {Object} container 真实的dom节点
 */
const mount = (vnode, container) => {
  const { tag, props, children } = vnode;
  // 1.创建真实元素，并且在vnode上保留
  const el = vnode.el = document.createElement(tag);
  // 2.判断props，设置att
  if(props) {
    for(let key in props) {
      const value = props[key];
      if(key.startsWith("on")) {
        el.addEventListener(key.slice(2).toLocaleLowerCase(), value);
      } else {
        el.setAttribute(key, value);
      }
    }
  }
  // 3.判断children
  if(children) {
    if(typeof children === "string") {
      el.textContent = children
    } else {
      children.forEach(element => {
        mount(element, el)
      });
    }
  }
  // 4.将虚拟dom挂载在真实dom上
  container.appendChild(el);
}

3、patch函数 ​

patch函数的功能是对diff算法的简单模拟，主要比对两个虚拟DOM节点是否有变化，有变化则更新，主要参数是oldNode(旧节点)、newNode（新节点）

1.两个节点的tag不一致，这种情况比较简单，直接拿到节点的父节点然后暴力替换就好了

js
if(n1.tag !== n2.tag) {
    const preEl = el.parentElement;
    preEl.removeChild(n1.el);
    mount(n2, preEl);
    return;
  }

2.两个节点一致，那我们就需要先处理props

js
// 2、处理props
  const oldProps = n1.props || {};
  const newProps = n2.props || {};
  // 2.1 获取所有新的props，与旧的不一致的就添加上
  for(let key in newProps) {
    const oldValue = oldProps[key];
    const newValue = newProps[key];
    if(oldValue !== newValue) { // 如果newProps中的属性的值和oldProps中的不一致那么便更新
      if(key.startsWith("on")) {
        el.removeEventListener(key.slice(2).toLocaleLowerCase(), oldValue); // js中函数是引用类型，每次声明地址都不一样，所以需要先取消监听
        el.addEventListener(key.slice(2).toLocaleLowerCase(), newValue);
      } else {
        el.setAttribute(key, newValue);
      }
    }
  }
  // 2.2删除旧的没有的props
  for(let key in oldProps) {
    if(!(key in newProps)) { // 如果oldProps中有newProps中没有的属性就删除
      if(key.startsWith("on")) {
        el.removeEventListener(key.slice(2).toLocaleLowerCase(), oldProps[key])
      } else {
        el.removeAttribute(key)
      }
    }
  }

3.接着就是处理children

js
// 3、处理children
  const oldChildren = n1.children || [];
  const newChildren = n2.children || [];
  // 3.1 新的children是字符串时直接替换
  if(typeof newChildren === "string") {
    if(typeof oldChildren === "string") {
      oldChildren !== newChildren ? el.textContent = newChildren : '';
    }else {
      el.innerHTML = newChildren
    }
  }else {
    // 3.2 新的children是数组
    if(typeof oldChildren === "string") {
      el.innerHTML = "";
      newChildren.forEach(item => mount(item, el));
    }else {
      const commonLength = Math.min(oldChildren.length, newChildren.length); // 找到最小的一个长度，处理两个节点相同长度部分的子节点
      // 3.2.2 替换数组长度一致的部分的内容
      for(let i = 0; i < commonLength; i++) {
        patch(oldChildren[i], newChildren[i]);
      }
      // 3.2.3 oldChildren > newChildren
      if(oldChildren.length > newChildren.length) {
        oldChildren.slice(commonLength).forEach(item => {
          el.removeChild(item.el);
        })
      }
      // 3.2.4 oldChildren < newChildren
      if(newChildren.length > oldChildren.length) {
        newChildren.slice(commonLength).forEach(item => {
          mount(item, el);
        })
      }
    }
  }

上面代码是较为简单的比较，子节点长度一致的部分是利用之前实现的patch做一个替换，长度不一致主要就是删除多的，添加新的，当然还有n1.children是字符串而n2.children是数组的情况也做了单独处理，但实际上还有很多边界情况没有处理，有机会可以继续完善，完整代码如下

js
/**
 * 对比两个节点
 * @param {Object} n1 旧的虚拟节点 
 * @param {Object} n2 新的虚拟节点
 * @returns 
 */
const patch = (n1, n2) => {
  const el = n2.el = n1.el;
  // 1、对比tag类型是否一致
  if(n1.tag !== n2.tag) {
    const preEl = el.parentElement;
    preEl.removeChild(n1.el);
    mount(n2, preEl);
    return;
  }

  // 2、处理props
  const oldProps = n1.props || {};
  const newProps = n2.props || {};
  // 2.1 获取所有新的props，与旧的不一致的就添加上
  for(let key in newProps) {
    const oldValue = oldProps[key];
    const newValue = newProps[key];
    if(oldValue !== newValue) {
      if(key.startsWith("on")) {
        el.removeEventListener(key.slice(2).toLocaleLowerCase(), oldValue);
        el.addEventListener(key.slice(2).toLocaleLowerCase(), newValue);
      } else {
        el.setAttribute(key, newValue);
      }
    }
  }
  // 2.2删除旧的没有的props
  for(let key in oldProps) {
    if(!(key in newProps)) {
      if(key.startsWith("on")) {
        el.removeEventListener(key.slice(2).toLocaleLowerCase(), oldProps[key])
      } else {
        el.removeAttribute(key)
      }
    }
  }

  // 3、处理children
  const oldChildren = n1.children || [];
  const newChildren = n2.children || [];
  // 3.1 新的children是字符串时直接替换
  if(typeof newChildren === "string") {
    if(typeof oldChildren === "string") {
      oldChildren !== newChildren ? el.textContent = newChildren : '';
    }else {
      el.innerHTML = newChildren
    }
  }else {
    // 3.2 新的children是数组
    if(typeof oldChildren === "string") {
      el.innerHTML = "";
      newChildren.forEach(item => mount(item, el));
    }else {
      const commonLength = Math.min(oldChildren.length, newChildren.length);
      // 3.2.2 替换数组长度一致的部分的内容
      for(let i = 0; i < commonLength; i++) {
        patch(oldChildren[i], newChildren[i]);
      }

      // 3.2.3 oldChildren > newChildren
      if(oldChildren.length > newChildren.length) {
        oldChildren.slice(commonLength).forEach(item => {
          el.removeChild(item.el);
        })
      }

      // 3.2.4 oldChildren < newChildren
      if(newChildren.length > oldChildren.length) {
        newChildren.slice(commonLength).forEach(item => {
          mount(item, el);
        })
      }
    }
  }
}

至此完成了miniVue的渲染模块，接下来是响应式的模块，vue的响应式重要的就在于依赖收集，老师的miniVue简单的实现了Dep类，用来收集依赖，订阅和通知

4、Dep类（收集依赖） ​

js
let activeEffect = null;
class Dep {
  constructor() {
    this.subscribe = new Set();
  }
  // 收集依赖
  depend() {
    if(activeEffect) {
      this.subscribe.add(activeEffect)
    }
  }
  notify() {
    this.subscribe.forEach(effcet => effcet());
  }
}
