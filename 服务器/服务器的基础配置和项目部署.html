三、安装nodejs ​

安装软件使用的工具：dnf

  • dnf全称dandified(时髦的、华丽的) Yum
  • 是yum的下一个版本，也被称之为yum的替代品
  • centos8自带dnf

检查dnf是否可用：

apl
dnf --help

如果我们希望安装一个软件包，可进行如下操作：

apl
# 搜索软件名
dnf search nodejs

# 查看软件包信息
dnf info nodejs

# 安装nodejs
dnf install nodejs -y

# 查看node是否安装成功和安装的版本
node -v

由于dnf能下载的版本较低，所以我们安装之后还需要进行版本升级

apl
# 下载nodejs版本管理工具
npm install -g n 
# 安装最新的lts(最新长期维护版)版本
n install lts
#下载某一版本号node  n 10.16.0
n <version> 
#删除某个版本   n  rm 10.16.0
n rm <version>  
#输入命令后直接使用上下箭头选择版本
n

如果版本切换不生效，退出重新登录即可

四、安装mysql ​

1、查看是否有mysql-server，centeos7是没有的 ​

apl
# 搜索软件名
dnf search mysql-server

# 查看软件包信息
dnf info mysql-server

# 安装nodejs
dnf install mysql-server -y

2、启动mysql-server ​

apl
# 开启MySQL后台服务
systemctl start mysqld
# 查看MySQL服务 active(running)表示启动成功
systemctl status mysql
# 随着系统一起启动
systemctl enable mysqld

3、配置mysql-server ​

由于dnf安装mysql-server时并未设置密码，所以我们需要做些配置，方便后面远程连接

apl
mysql_secure_installation

# 接下来会出现一些选项，重点在于选择密码强度，mysql8.0对密码强度要求较高，大概需要特殊字符和数字和大小写字母分别都有

我在Disallow root login remotely选择时填了n,允许远程访问

4、使用Navicat远程连接MySQL ​

有几个重点：

  • 服务器的安全组需要加入3306的端口
  • 主机需要填服务器的公网ip
  • 设置MySQL的root用户接受远程连接

进入mysql服务，并切换到mysql数据库，查看uesr表

apl
mysql -u root -p
# 进入mysql数据库
use mysql;
# 查看user表的host、user字段
select host, user from user;

默认只能lcoalhost本机连接，我们要更改为所有都能连接(酌情配置，此操作有被攻击的风险)

apl
update user set host = '%' where user = 'root';
# 配置后保险起见，刷新一下
flush privileges;

然后就能进行远程连接了，连接名随意

五、上传文件到服务器 ​

1、上传文件 ​

至此基础环境已经具备了，接下来就可以上传文件让服务器运行

上传文件主要有三种方式

  • 使用工具直接拷贝到服务器
  • 将文件上传到git仓库，远程服务器再git clone
  • 使用自动化部署，例如jenkins

本文主要使用上传到git仓库的方式

2、将代码上传到自己Git仓库，这里以github为例 ​

  • 通过填写**.gitignore文件**忽略一些私人配置文件和node_moduls

3、将代码clone到服务器 ​

  • 创建存放代码的文件夹
apl
# 进入根目录
cd ..
# 安装git工具
dnf install git -y
# 执行clone命令
git clone 仓库地址
# 可能会报错，执行以下命令即可
git config --global http.sslVerify "false"
# 产生原因：一般是这是因为服务器的SSL证书没有经过第三方机构的签署，所以才报错

4、运行代码 ​

pm2介绍：

PM2是一个用于Node.js应用程序的生产过程管理器，具有内置的负载均衡器。它允许您让应用程序永远处于活动状态，在没有停机时间的情况下重新加载它们，并方便执行常见的系统管理任务。

本文用到的是pm2 进程管理，nodejs项目一直运行，仓库看看

介绍完工具后，我们开始在服务器上运行代码

apl
# 安装依赖
npm install 
# 启动应用
pm2 start main.js --name 应用名称(自定义)
# 监控应用目录，一旦有文件发生更改就立刻重启应用
pm2 start app.js --watch
# 重启
pm2 restart app_name|app_id|all
# 重载
pm2 reload app_name|app_id|all
# 停止
pm2 stop app_name|app_id|all
# 删除
pm2 delete app_name|app_id|all
# 查看运行状态
pm2 [list|ls|status]
